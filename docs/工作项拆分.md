# 农产品融销一体平台工作项拆分

本文档参考当前项目管理中的 Epic 模块，将后端相关工作拆分为可执行的 Story/Task，便于录入 Backlog、分配开发任务和验收。

## 1. 系统管理员模块

### Story 1.1 管理员登录与权限校验

- 类型: Story
- 优先级: 高
- 预估工时: 5 小时
- 目标: 系统管理员可以通过账号登录后台，并访问管理员专属接口。
- 主要接口:
  - `POST /api/auth/login`
  - `GET /api/system/roles`
  - `GET /api/admin/overview`
- 验收标准:
  - 管理员账号登录成功后返回用户信息、角色和访问标识。
  - 非管理员角色访问 `/api/admin/**` 时返回无权限。
  - 管理员可以正常访问后台概览数据。

### Task 1.2 用户管理接口

- 类型: Task
- 优先级: 高
- 预估工时: 2 小时
- 目标: 管理员可以查看所有用户并调整业务角色。
- 主要接口:
  - `GET /api/admin/users`
  - `PATCH /api/admin/users/{userName}/role`
  - `GET /api/users`
  - `GET /api/users/{userName}`
- 验收标准:
  - 支持查询用户列表和用户详情。
  - 支持修改普通业务用户角色。
  - 不允许普通接口创建系统管理员账号。

### Task 1.3 商品货源审核与上下架管理

- 类型: Task
- 优先级: 高
- 预估工时: 2 小时
- 目标: 管理员可以查看并处理农户发布的货源商品状态。
- 主要接口:
  - `GET /api/admin/trade/orders`
  - `PATCH /api/admin/trade/orders/{orderId}/status`
  - `GET /api/trade/orders`
  - `PATCH /api/trade/orders/{orderId}/status`
- 验收标准:
  - 管理员可以查看全部货源。
  - 管理员可以调整货源状态。
  - 状态变更后前台查询结果同步更新。

### Task 1.4 采购订单审核与管理

- 类型: Task
- 优先级: 中
- 预估工时: 2 小时
- 目标: 管理员可以查看采购订单并处理订单状态。
- 主要接口:
  - `GET /api/admin/trade/purchases`
  - `PATCH /api/admin/trade/purchases/{purchaseId}/status`
- 验收标准:
  - 管理员可以查看全部采购订单。
  - 管理员可以变更采购订单状态。
  - 修改后的状态可被买家个人订单接口查询到。

### Task 1.5 融资申请审批

- 类型: Task
- 优先级: 高
- 预估工时: 2 小时
- 目标: 管理员可以查看融资申请并进行审批。
- 主要接口:
  - `GET /api/admin/finance/applications`
  - `PATCH /api/admin/finance/applications/{financeId}/status`
- 验收标准:
  - 管理员可以查看全部融资申请。
  - 支持通过、拒绝或备注审批结果。
  - 审批结果可在融资申请详情中查询。

## 2. 买家模块

### Story 2.1 买家注册、登录与基本信息维护

- 类型: Story
- 优先级: 高
- 预估工时: 5 小时
- 目标: 买家可以完成注册、登录和个人资料维护。
- 主要接口:
  - `POST /api/auth/register`
  - `POST /api/auth/login`
  - `GET /api/users/{userName}`
  - `PUT /api/users/{userName}`
  - `PATCH /api/users/{userName}/password`
- 验收标准:
  - 买家可以注册并以买家角色登录。
  - 买家可以查看和更新个人信息。
  - 买家可以修改登录密码。

### Task 2.2 商品货源查询

- 类型: Task
- 优先级: 高
- 预估工时: 2 小时
- 目标: 买家可以浏览农户发布的商品货源。
- 主要接口:
  - `GET /api/trade/orders`
  - `GET /api/trade/orders/{orderId}`
- 验收标准:
  - 支持查询货源列表。
  - 支持查看单个货源详情。
  - 返回字段包含标题、价格、图片、地址、发布人和状态。

### Task 2.3 购物车管理

- 类型: Task
- 优先级: 中
- 预估工时: 2 小时
- 目标: 买家可以将商品加入购物车并维护数量。
- 主要接口:
  - `GET /api/trade/shopping-cart`
  - `GET /api/trade/shopping-cart/owners/{ownName}`
  - `POST /api/trade/shopping-cart`
  - `PUT /api/trade/shopping-cart/{shoppingId}`
  - `DELETE /api/trade/shopping-cart/{shoppingId}`
- 验收标准:
  - 买家可以新增购物车记录。
  - 买家可以查询自己的购物车。
  - 买家可以修改数量或删除购物车项。

### Task 2.4 采购订单生成与查询

- 类型: Task
- 优先级: 高
- 预估工时: 2 小时
- 目标: 买家可以根据商品生成采购订单，并查看个人采购记录。
- 主要接口:
  - `POST /api/trade/purchases`
  - `GET /api/trade/purchases/owners/{ownName}`
  - `GET /api/trade/purchases/{purchaseId}`
  - `GET /api/trade/purchases/{purchaseId}/details`
- 验收标准:
  - 买家可以提交采购订单。
  - 订单详情包含商品明细、数量、单价和总价。
  - 买家可以查询自己的采购订单列表。

### Task 2.5 收货地址管理

- 类型: Task
- 优先级: 中
- 预估工时: 1 小时
- 目标: 买家可以维护个人收货地址。
- 主要接口:
  - `GET /api/addresses/owners/{ownName}`
  - `POST /api/addresses`
  - `PUT /api/addresses/{id}`
  - `DELETE /api/addresses/{id}`
- 验收标准:
  - 买家可以新增、修改、删除地址。
  - 支持按用户查询地址列表。
  - 地址字段包含收货人、手机号、详细地址和默认标识。

## 3. 银行模块

### Story 3.1 银行用户登录与银行产品管理

- 类型: Story
- 优先级: 高
- 预估工时: 5 小时
- 目标: 银行用户可以登录系统并维护贷款产品。
- 主要接口:
  - `POST /api/auth/login`
  - `GET /api/finance/banks`
  - `GET /api/finance/banks/{bankId}`
  - `POST /api/finance/banks`
  - `PUT /api/finance/banks/{bankId}`
  - `DELETE /api/finance/banks/{bankId}`
- 验收标准:
  - 银行用户可以登录银行模块。
  - 支持贷款产品的新增、查询、修改和删除。
  - 产品字段包含银行名称、金额、利率、期限和联系方式。

### Task 3.2 融资申请查询与处理

- 类型: Task
- 优先级: 高
- 预估工时: 2 小时
- 目标: 银行可以查看农户融资申请并更新处理状态。
- 主要接口:
  - `GET /api/finance/applications`
  - `GET /api/finance/applications/{financeId}`
  - `PATCH /api/finance/applications/{financeId}/status`
- 验收标准:
  - 银行可以查看融资申请列表和详情。
  - 银行可以更新申请状态和备注。
  - 更新结果对农户侧查询可见。

### Task 3.3 银行侧智能匹配农户

- 类型: Task
- 优先级: 中
- 预估工时: 2 小时
- 目标: 银行可以根据贷款产品匹配有融资意向的农户。
- 主要接口:
  - `GET /api/finance/matches/farmers/{bankId}`
- 验收标准:
  - 根据银行产品金额、利率或期限匹配融资意向。
  - 返回农户融资意向列表。
  - 匹配结果包含农户用户名、申请用途、金额和还款周期。

### Task 3.4 融资申请审批后台联动

- 类型: Task
- 优先级: 中
- 预估工时: 2 小时
- 目标: 银行处理结果与系统管理员审批结果保持一致。
- 主要接口:
  - `GET /api/admin/finance/applications`
  - `PATCH /api/admin/finance/applications/{financeId}/status`
- 验收标准:
  - 管理员可查看银行处理后的融资申请状态。
  - 状态字段含义统一。
  - 异常状态更新返回明确错误信息。

## 4. 专家模块

### Story 4.1 专家信息管理

- 类型: Story
- 优先级: 高
- 预估工时: 4 小时
- 目标: 专家可以维护个人专家资料，用户可以查询专家列表。
- 主要接口:
  - `GET /api/experts`
  - `GET /api/experts/{userName}`
  - `POST /api/experts`
  - `PUT /api/experts/{userName}`
  - `DELETE /api/experts/{userName}`
- 验收标准:
  - 支持专家列表和详情查询。
  - 支持专家资料新增、修改和删除。
  - 专家资料包含专业方向、真实姓名、所属单位和联系方式。

### Task 4.2 农业知识发布与管理

- 类型: Task
- 优先级: 高
- 预估工时: 2 小时
- 目标: 专家可以发布和维护农业知识内容。
- 主要接口:
  - `GET /api/knowledge`
  - `GET /api/knowledge/{knowledgeId}`
  - `POST /api/knowledge`
  - `PUT /api/knowledge/{knowledgeId}`
  - `DELETE /api/knowledge/{knowledgeId}`
- 验收标准:
  - 支持知识文章的新增、查询、修改和删除。
  - 返回字段包含标题、正文、图片路径和发布人。
  - 删除知识后对应详情不可继续查询。

### Task 4.3 农业知识讨论管理

- 类型: Task
- 优先级: 中
- 预估工时: 2 小时
- 目标: 用户可以对农业知识进行评论讨论。
- 主要接口:
  - `GET /api/knowledge/{knowledgeId}/discusses`
  - `POST /api/knowledge/{knowledgeId}/discusses`
  - `DELETE /api/knowledge/{knowledgeId}/discusses/{discussId}`
- 验收标准:
  - 支持按知识文章查询评论列表。
  - 支持新增评论。
  - 支持删除指定评论。

### Task 4.4 在线农业问答处理

- 类型: Task
- 优先级: 高
- 预估工时: 2 小时
- 目标: 专家可以查看农户提问并进行答复。
- 主要接口:
  - `GET /api/consultation/questions`
  - `GET /api/consultation/questions/{id}`
  - `PATCH /api/consultation/questions/{id}/answer`
- 验收标准:
  - 专家可以查看问题列表和详情。
  - 专家可以提交答复内容。
  - 答复后问题状态更新。

### Task 4.5 专家预约处理

- 类型: Task
- 优先级: 中
- 预估工时: 2 小时
- 目标: 专家可以查看和处理农户预约。
- 主要接口:
  - `GET /api/consultation/reserves`
  - `GET /api/consultation/reserves/{id}`
  - `PATCH /api/consultation/reserves/{id}/answer`
- 验收标准:
  - 专家可以查看预约列表和详情。
  - 专家可以回复预约。
  - 预约状态随回复同步更新。

## 5. 农户模块

### Story 5.1 农户注册、登录与个人中心

- 类型: Story
- 优先级: 高
- 预估工时: 5 小时
- 目标: 农户可以注册登录并维护个人基本信息。
- 主要接口:
  - `POST /api/auth/register`
  - `POST /api/auth/login`
  - `GET /api/users/{userName}`
  - `PUT /api/users/{userName}`
  - `PATCH /api/users/{userName}/password`
- 验收标准:
  - 农户可以注册并以农户角色登录。
  - 农户可以查看和修改个人信息。
  - 农户可以修改密码。

### Task 5.2 个人货源管理

- 类型: Task
- 优先级: 高
- 预估工时: 2 小时
- 目标: 农户可以发布、查询、修改和删除个人货源。
- 主要接口:
  - `GET /api/trade/orders/owners/{ownName}`
  - `POST /api/trade/orders`
  - `PUT /api/trade/orders/{orderId}`
  - `DELETE /api/trade/orders/{orderId}`
- 验收标准:
  - 农户可以发布商品货源。
  - 农户可以查询自己的货源列表。
  - 农户可以修改或删除自己的货源。

### Task 5.3 销售订单查询

- 类型: Task
- 优先级: 中
- 预估工时: 2 小时
- 目标: 农户可以查看买家采购自己商品形成的销售订单。
- 主要接口:
  - `GET /api/trade/orders/cooperators/{cooperationName}`

  - 返回订单状态、买家信息和商品信息。
  - 订单状态与采购订单处理结果一致。

### Task 5.4 农业知识查询与讨论

- 类型: Task
- 优先级: 中
- 预估工时: 2 小时
- 目标: 农户可以浏览农业知识并参与讨论。
- 主要接口:
  - `GET /api/knowledge`
  - `GET /api/knowledge/{knowledgeId}`
  - `GET /api/knowledge/{knowledgeId}/discusses`
  - `POST /api/knowledge/{knowledgeId}/discusses`
- 验收标准:
  - 农户可以查询知识列表和详情。
  - 农户可以查看评论列表。
  - 农户可以发布讨论内容。

### Task 5.5 在线农业问答

- 类型: Task
- 优先级: 中
- 预估工时: 2 小时
- 目标: 农户可以向专家提交农业问题并查看答复。
- 主要接口:
  - `POST /api/consultation/questions`
  - `GET /api/consultation/questions`
  - `GET /api/consultation/questions/{id}`
- 验收标准:
  - 农户可以提交问题。
  - 问题字段包含专家、提问人、联系方式、作物、标题和问题描述。
  - 专家答复后农户可以查看答案和状态。

### Task 5.6 专家预约申请

- 类型: Task
- 优先级: 中
- 预估工时: 2 小时
- 目标: 农户可以预约专家进行线下或线上指导。
- 主要接口:
  - `POST /api/consultation/reserves`
  - `GET /api/consultation/reserves`
  - `GET /api/consultation/reserves/{id}`
- 验收标准:
  - 农户可以提交预约信息。
  - 预约字段包含专家、面积、地址、作物、土壤情况、种植情况和留言。
  - 专家处理后农户可以查看预约结果。

### Task 5.7 融资意向管理

- 类型: Task
- 优先级: 高
- 预估工时: 2 小时
- 目标: 农户可以维护个人融资意向。
- 主要接口:
  - `GET /api/finance/intentions`
  - `GET /api/finance/intentions/{id}`
  - `POST /api/finance/intentions`
  - `PUT /api/finance/intentions/{id}`
  - `DELETE /api/finance/intentions/{id}`
- 验收标准:
  - 农户可以新增融资意向。
  - 农户可以查看、修改和删除融资意向。
  - 融资意向字段包含用途、项目、金额和还款周期。

### Task 5.8 个人智能匹配银行

- 类型: Task
- 优先级: 高
- 预估工时: 4 小时
- 目标: 农户可以根据融资金额匹配合适的银行贷款产品。
- 主要接口:
  - `GET /api/finance/banks/matches?amount={amount}`
- 验收标准:
  - 输入融资金额后返回匹配银行列表。
  - 匹配结果包含银行名称、可贷金额、利率和联系方式。
  - 未匹配到产品时返回空列表，不返回系统错误。

### Task 5.9 融资申请提交与查询

- 类型: Task
- 优先级: 高
- 预估工时: 2 小时
- 目标: 农户可以向银行提交融资申请并查看处理结果。
- 主要接口:
  - `POST /api/finance/applications`
  - `GET /api/finance/applications`
  - `GET /api/finance/applications/{financeId}`
- 验收标准:
  - 农户可以提交融资申请。
  - 申请字段包含银行、申请人、身份证号、金额、利率、还款方式和附件信息。
  - 银行或管理员审批后农户可以查看状态。

### Task 5.10 图片上传

- 类型: Task
- 优先级: 中
- 预估工时: 2 小时
- 目标: 农户发布货源、知识或融资材料时可以上传图片。
- 主要接口:
  - `POST /api/files/images`
- 验收标准:
  - 支持 multipart 图片上传。
  - 返回原始文件名、文件类型、访问地址等信息。
  - 非图片类型或空文件返回明确错误。

## 6. 公共质量与联调任务

### Task 6.1 接口统一响应格式

- 类型: Task
- 优先级: 高
- 预估工时: 2 小时
- 目标: 所有接口统一返回 `success`、`code`、`message`、`data` 结构。
- 验收标准:
  - 成功响应结构一致。
  - 业务异常和权限异常结构一致。
  - 前端可根据 `code` 和 `success` 统一处理。

### Task 6.2 接口权限矩阵校验

- 类型: Task
- 优先级: 高
- 预估工时: 2 小时
- 目标: 按买家、农户、专家、银行、系统管理员划分接口访问权限。
- 验收标准:
  - 未登录访问受保护接口返回 401。
  - 角色不匹配返回 403。
  - 公共查询接口可按设计开放访问。

### Task 6.3 OpenAPI 接口文档

- 类型: Task
- 优先级: 中
- 预估工时: 2 小时
- 目标: 所有后端接口可以通过 OpenAPI 文档查看。
- 主要接口:
  - `GET /v3/api-docs`
  - `/swagger-ui/**`
- 验收标准:
  - 文档包含所有已实现接口路径。
  - 接口请求方法、路径参数和请求体可被识别。
  - 文档可用于前后端联调。

### Task 6.4 后端自动化测试

- 类型: Task
- 优先级: 高
- 预估工时: 2 小时
- 目标: 使用 MockMvc 覆盖核心业务接口。
- 验收标准:
  - 覆盖登录、权限、用户、货源、采购、知识、专家、融资、文件上传等接口。
  - 测试数据使用测试环境数据库。
  - 执行 `mvnw.cmd test` 后全部通过。
